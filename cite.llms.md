# Citing ACQUIRE

## Cite the artifact

ACQUIRE is released as a versioned, citable artifact — this website and the taxonomy, checklist and templates it publishes. Cite it by version, because thresholds and recipes change as evidence accumulates and a version identifies exactly what a study was assessed against. A bare URL does not.

``` bibtex
@software{acquire_2026,
  author  = {Danio{\l}, Mateusz and Sroka, Ryszard},
  title   = {{ACQUIRE}: Acquisition Criteria for Quality, Uncertainty,
             Integrity, Reproducibility, and Evidence},
  version = {26.7.2},
  year    = {2026},
  url     = {https://acquire-framework.github.io/v26.7.2/}
}
```

A DOI will be minted on the first archived deposit and added here. Until then, the versioned permalink above is the stable reference.

## The checklist

If you attach a completed checklist as supplementary material, state the checklist version explicitly — it is stamped in the exported artifact:

> Data collection followed the ACQUIRE Minimum Reporting Checklist, specification 26.7.2 (<https://acquire-framework.github.io/v26.7.2/>). The completed checklist, including justifications for items marked not applicable, is provided as supplementary material.

## A note on the manuscript

A manuscript describing the framework and its motivation is **in preparation**. It has not been published or accepted, so there is nothing yet to cite as a paper — please cite the artifact above. A paper reference will be added here if and when the manuscript is published, and the two will then be complementary: the paper for the argument, the artifact for the versioned content a study was assessed against.

## Contributing changes the citation

Contributors of accepted recipes, replications, and adapters are credited on the archived release record. This is deliberate: replication is the contribution the catalogue most needs, and it should carry credit that appears on a CV.

Contributing to the framework is distinct from co-authorship on any associated manuscript — see [CONTRIBUTING](https://github.com/acquire-framework/acquire-framework.github.io/blob/main/CONTRIBUTING.md) for how the two are handled.

``` quarto-appendix-bibtex
@software{acquire_2026,
  author = {Danioł, Mateusz and Sroka, Ryszard},
  title = {ACQUIRE: {Acquisition} {Criteria} for {Quality,}
    {Uncertainty,} {Integrity,} {Reproducibility,} and {Evidence}},
  version = {26.7.2},
  date = {2026},
  url = {https://acquire-framework.github.io},
  langid = {en}
}
```

For attribution, please cite this work as:

Danioł, Mateusz, and Ryszard Sroka. 2026. *ACQUIRE: Acquisition Criteria for Quality, Uncertainty, Integrity, Reproducibility, and Evidence*. V. 26.7.2. Released. <https://acquire-framework.github.io>.
